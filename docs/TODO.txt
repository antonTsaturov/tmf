уязвимости:

    Отсутствие аутентификации на основных API endpoints документов — любой пользователь может создавать, читать, изменять и удалять документы.
    Возможная SQL-инъекция через динамическую интерполяцию в условиях запроса.
    Отсутствие проверки прав доступа (роли пользователя) в защищенных endpoint'ах.
    Command injection в сервисе резервного копирования.
    Небезопасные заголовки CORS с 'unsafe-eval' и 'unsafe-inline'.
    Потенциальная утечка данных через агрегацию информации о документах.

Рекомендации:

    Добавить проверки аутентификации и авторизации на все критические endpoints.
    Использовать безопасные параметризованные SQL-запросы.
    Заменить небезопасные функции выполнения команд.
    Усилить заголовки безопасности (CSP).
    Приложение требует срочного рефакторинга механизмов безопасности.
