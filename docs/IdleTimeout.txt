Время бездействия (Idle Timeout)
Общая схема:
text

Клиентская активность → Обновление таймера → 30 минут бездействия → Выход

Конкретные цифры:

    Клиентский таймер (useIdleTimeout): 30 минут

    Серверный таймер (SESSION_CONFIG.IDLE_TIMEOUT): 30 минут

    Access Token: 15 минут (обновляется автоматически)

    Refresh Token: 7 дней

Что считается активностью?
На клиенте (события, которые сбрасывают таймер):
javascript

const ACTIVITY_EVENTS = [
  'mousemove',    // ✅ Движение мышью
  'mousedown',    // ✅ Клик мышью
  'keydown',      // ✅ Нажатие клавиши
  'scroll',       // ✅ Скролл страницы
  'touchstart',   // ✅ Касание на тачскрине
  'wheel',        // ✅ Вращение колесика мыши
  'click',        // ✅ Клик (дополнительно)
  'focus',        // ✅ Фокус на странице/элементе
];

Ответ на ваш вопрос: Да, если пользователь просто водит мышкой - таймер будет сбрасываться. Scroll, клики, движение - всё это активность.
На сервере (обновление сессии):
typescript

// Сессия обновляется при:
1. Любом API запросе (кроме /api/auth/refresh)
2. Обновлении activity через check-auth.ts
3. Успешном refresh (согласно новой логике)

Полный сценарий работы системы:
1. Нормальная работа (пользователь активен)
text

Временная линия:
00:00 - Пользователь залогинился (Access Token выдан на 15 мин)
00:05 - Пользователь нажал кнопку (мышкой поводил) → обновилась last_activity
00:10 - Пользователь скроллит → обновилась last_activity
00:14 - Access Token скоро истечет
00:14 - Фронт отправляет /api/auth/refresh (обновляет сессию)
00:29 - Token обновлен, сессия жива
00:30 - ... цикл повторяется

2. Пользователь бездействует (не двигает мышью)
text

00:00 - Последнее действие пользователя (клик, движение мыши)
05:00 - Пользователь смотрит видео (без движения мыши) → таймер не сбрасывается
15:00 - Access Token истек
15:00 - Фронт пытается обновить токен
15:00 - Refresh успешен, но last_activity НЕ обновляется
30:00 - last_activity была 30 минут назад (в 00:00)
30:00 - Серверная проверка: idleTime = 30 минут
30:00 - Сессия признается истекшей
30:01 - Следующий запрос получит 401
30:01 - Пользователь перенаправляется на логин

3. Предупреждение за 1 минуту до выхода
text

29:00 - Клиент видит: idleTime = 29 минут (осталась 1 минута)
29:00 - Показывается модальное окно: "Session expires in 60 seconds"
29:30 - Пользователь нажимает "Stay Logged In" → отправляется /api/auth/refresh
29:31 - Сессия продлена, таймер сброшен
30:00 - Сессия остается активной (была продлена)
